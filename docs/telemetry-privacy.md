# Telemetry Privacy — Level 4 Real-Usage Capture

Date: 2026-07-07
Status: Active

---

## What This Is

`scripts/telemetry/` is a set of Claude Code hooks that record real skill/agent
usage — invocation counts, whether the user corrected or discarded the output,
and which artifacts were produced — so `scripts/telemetry/aggregate-usage.js`
can advisorily flag skills/agents worth re-running through `skill-refine` /
`agent-refine`. It is Level 4 in the eval architecture's signal stack (see
`docs/codex-external-eval-architecture.md`), alongside native eval (Level 1),
cold-prediction Codex eval (Level 2 — replaced with) and native-audit mode
(Level 3).

Nothing in this system calls `skill-refine`, `agent-refine`, or any other
lifecycle step automatically. It only writes a `real_usage` block that a
human or a refine agent picks up.

## What Is Captured

Every event is one JSONL line in `evals/telemetry/events-YYYY-MM.jsonl`
(schema: `schemas/telemetry/invocation-event.schema.json`). A line may contain:

- **Timestamps** — `ts` (ISO-8601 UTC)
- **Identifiers** — `session_id` (Claude Code's own session id), `invocation_id`
  (a UUID generated by the hook, never derived from content)
- **Enums** — `kind` (`skill`/`agent`), `event` (`invocation`/`outcome`/`session_end`),
  `user_disposition` (`accepted`/`revised`/`rejected`/`unknown`), `confidence`
  (`low`/`medium`/`high`)
- **Booleans** — `followed_by_correction`, per-artifact `exists`
- **Counts / durations** — `duration_ms` when derivable from the hook payload
- **Repo-relative paths only** — `artifacts[]`, `artifact_census[].path`. Any
  path that resolves outside the repo root is dropped, not stored (see
  `redact.toRepoRelative` in `scripts/telemetry/redact.js`)
- **A one-way hash** — `context_hash`, the sha256 hex digest of whatever
  prompt-like text a given event derives from (a Task's `prompt` field, or a
  `UserPromptSubmit` prompt). This is the *only* representation of prompt text
  that is ever written.

## What Is NEVER Captured

- **Raw prompt text.** `log-outcome.js` reads the `UserPromptSubmit` prompt
  in-memory only, to run the correction heuristic and compute `context_hash`.
  The text itself is discarded once the hook process exits; it is never
  written to disk.
- **Transcript text.** Hooks never read or store `transcript_path` contents.
- **File contents.** Artifact tracking is existence/path-only (`fs.existsSync`),
  never a read of the file's contents.
- **Absolute or out-of-repo paths.** `redact.js` converts every path to
  repo-relative and silently drops anything that doesn't resolve inside the
  repo root (defends against leaking home-directory or system paths).
- **Secrets.** Any string that *is* stored (name, path) is passed through
  `redact.redactString()`, which regex-scans for common secret shapes
  (Anthropic/OpenAI/GitHub/Slack/AWS/Google API keys, PEM private keys,
  JWT-shaped tokens, bearer tokens, generic `key=`/`token=` assignments) and
  replaces matches with `[REDACTED]` as defense-in-depth — this is a backstop,
  not the reason prompt text is safe; prompt text is safe because it is never
  written at all.

## How the Correction Heuristic Works (and its limits)

`log-outcome.js` (`UserPromptSubmit`, `--mode correction`) fires **only** when
both of these hold for the newest prompt in the session:

1. It mentions the basename of an artifact produced by the session's most
   recent invocation (case-insensitive substring match), **and**
2. It contains revision language: `fix`/`fixed`/`fixing`, `redo`, `wrong`,
   `instead`, `actually`, `change`/`changed`, or the phrase `no,` (word-boundary
   guarded so it doesn't match inside words like "casino,").

This is a heuristic, not ground truth — a documented risk (see the plan's
"Risks / open points": *"Correction-detection heuristic will misclassify"*).
Mitigations:

- `confidence` (`low`/`medium`) is attached to every fired outcome event —
  downstream consumers should weight accordingly, never treat it as certain.
- The aggregator's refine-trigger policy requires **≥10 invocations in the
  trailing 30 days** before acting on any rate computed from this heuristic —
  a single misfire can't flip a target to `REFINE_RECOMMENDED`.
- `rejection_rate` is derived independently, from `session_end` artifact
  census (an artifact recorded by an invocation that no longer exists on disk
  at session close) — not from prompt-text pattern matching.

## Retention

- One JSONL file per calendar month: `evals/telemetry/events-YYYY-MM.jsonl`.
- `scripts/telemetry/aggregate-usage.js` prunes down to the most recent 6
  monthly files every time it runs. Rotation lives in the aggregator, not the
  hooks — hooks only ever append to the current month's file.
- `evals/telemetry/usage-summary.json` is fully regenerated on each aggregator
  run (not append-only).
- Nothing under `evals/telemetry/` is committed — see `evals/.gitignore`.

## How to Disable

- **Stop capture immediately:** remove the four hook entries from
  `.claude/settings.json` under `hooks.PostToolUse` (matcher `Skill|Task`),
  `hooks.UserPromptSubmit`, `hooks.Stop`, and `hooks.SessionEnd` that point at
  `scripts/telemetry/log-invocation.js` / `log-outcome.js`. `update-config`-style
  edits are safe — the hooks are self-contained and fail open.
- **Full removal:** run `./uninstall.sh /path/to/target-project` — it removes
  `scripts/telemetry/`, `schemas/telemetry/`, and surgically un-merges only the
  four hook entries it added, leaving every other hook/setting untouched (see
  uninstall.sh step 7).
- **Delete history without disabling capture:** delete files under
  `evals/telemetry/` directly; the hooks recreate the directory and current
  month's file on the next invocation.

## Threat Notes

- **Fail-open by design.** Every hook wraps its entire body in try/catch,
  enforces a hard timeout well under the hook budget, and always exits 0 —
  a telemetry failure must never block or fail the tool call it observed.
  This means a corrupted or unreadable JSONL file degrades the aggregator's
  output, not the user's session.
- **Corrupt/partial lines are skipped, not fatal.** Both hooks and the
  aggregator parse JSONL line-by-line and discard unparseable lines rather
  than aborting the whole read.
- **No cross-repo leakage.** `CLAUDE_PROJECT_DIR` (or the hook's own `cwd`
  field) scopes every read/write to one repo's `evals/telemetry/`; artifact
  paths outside that root are dropped at capture time, not merely redacted.
- **Advisory only, never authoritative.** `REFINE_RECOMMENDED` is a flag in a
  JSON file a human or `skill-refine`/`agent-refine` reads — this system has no
  code path that invokes those lifecycle steps itself.
