## What Eve is
Vercel open-source, **filesystem-first** agent framework. Principle: *"a file's location says what it
doesвЂ¦ adding a file automatically registers it"* вЂ” the directory layout is the registry, no separate
config. TypeScript (~96%), Node-based, Claude-configurable. Beta (Vercel beta terms). Durable sessions
powered by the open-source Workflow SDK. Repo signals at capture time: ~3.7k stars, 60 releases.

## Built-in features (as documented)
- **Durable sessions** вЂ” more than one request/response; streaming, tool calls, human-approval pauses,
  resumption (Workflow SDK).
- **Human-in-the-loop** вЂ” pause for approval/answer, then resume.
- **Sandboxed execution** вЂ” controlled file/command workspace (Vercel Sandbox).
- **Multi-platform portability** вЂ” same behavior across web/terminal/Slack.
- **Subagents / connections** вЂ” team hierarchies; external MCP/OpenAPI tools.

## eve-analyst (production-shaped reference agent, coleam00)
- Layout: `agent/` (agent.ts via `defineAgent`, instructions.md, tools/ = list_tables/describe_table/
  run_sql/run_analysis, skills/ e.g. revenue-rules.md, subagents/investigator/, channels/ = eve.ts +
  slack.ts, lib/ = db.ts/sql-guard.ts/sandbox.ts), plus `evals/` (deploy gate), `test/`, `.claude/`.
- Stack: TypeScript(~70%)/JS(~30%), Node 24+, pnpm 10+; `pnpm-workspace.yaml`, `tsconfig.json`,
  `.vercelignore`, `.env.local`.
- Controls demonstrated: read-only SQL guard (blocks non-SELECT), HITL approval on expensive queries,
  durable pause/resume, Python-in-sandbox with `networkPolicy: "deny-all"`, autonomous subagent.
