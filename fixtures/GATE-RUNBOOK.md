# GATE-RUNBOOK — Phase 3 live calibration gate (human-invoked)

The exact command sequence a human runs to execute the Phase 3 gate: prove the native eval +
Codex native-audit layers together catch all 4 injected defects in
`fixtures/mutant-brief-writer/SKILL.md`. Only step 3 spends API credits — every other step is
dry-run-safe. **Never automate step 3; `--live` is always typed by a human** (binding CLAUDE.md
rule).

Ground truth: `fixtures/mutant-brief-writer/expected-findings.json`. Do not edit the mutant or the
manifest to make the gate pass — if a defect is missed, iterate on the catching layer (e.g.
`buildPrompt()` in `scripts/codex/run-native-audit.js`), then re-run steps 3–4. The fixture makes
that loop cheap and regression-safe.

---

## Step 1 — Generate scenarios (no API cost)

```bash
node scripts/run-calibration.js generate
```

Expected output: 9 scenarios written to `evals/fixtures/mutant-brief-writer/evals.json`, prompts
naming the concrete golden target (`changelog-entry`), and a per-scenario echo-lint report.

- Echo lint PASS is the current verified behavior (prompts are target-templated, so they don't
  echo the mutant's description even though it's vague).
- If the lint ever FLAGS scenarios: that is discrimination signal from the deliberately vague
  description, not an orchestration error — the script reports it and continues where possible.
  Record it in the gate results either way.

## Step 2 — Native eval on the mutant via skill-eval-agent (no Codex cost; normal Claude session cost)

`skill-eval-agent`'s definition resolves everything from `skills/<skill-name>/...` by default.
The mutant must **never** be copied or symlinked into `skills/` — not even temporarily (it would
be picked up by dynamic skill discovery, `install.sh`, and the `~/.claude` deploy rule). Instead,
dispatch the agent with explicit path overrides in the task prompt.

Dispatch `skill-eval-agent` (Agent tool / Task) with a prompt equivalent to:

```
Evaluate the skill defined at fixtures/mutant-brief-writer/SKILL.md (skill name:
mutant-brief-writer). This is a calibration fixture living OUTSIDE skills/ — use these
explicit paths instead of your default skills/<skill-name>/ resolution, and NEVER copy
the fixture into skills/ or edit it:

- Definition file:    fixtures/mutant-brief-writer/SKILL.md  (read-only)
- evals.json:         evals/fixtures/mutant-brief-writer/evals.json
                      (ALREADY GENERATED by run-calibration.js — do not regenerate;
                      skip your scenario-generation step)
- Iteration root:     evals/fixtures/mutant-brief-writer/iteration-1/
                      (run-manifest.json, s<id>-<type>-r<rep>/ scenario dirs,
                      per-scenario workspace/ sandboxes, evidence.json — all your usual
                      artifacts, rooted here instead of evals/mutant-brief-writer/)
- Report output:      fixtures/mutant-brief-writer/SKILL-EVAL.md
                      (next to the definition — required by run-native-audit.js
                      --def-path convention; this path is gitignored)
- Baseline method:    none (new skill, no prior SKILL-EVAL.md)
- Harvest evidence with:
  node skills/skill-eval/scripts/harvest-evidence.js evals/fixtures/mutant-brief-writer/iteration-1 --type skill --all
- If metrics fall below threshold, write refine-input.json under
  evals/fixtures/mutant-brief-writer/ but DO NOT invoke skill-refine — this fixture is
  never refined (fixtures/README.md immutability rule).
```

Sanity checks after it finishes:

```bash
node skills/skill-eval/scripts/run-manifest.js status evals/fixtures/mutant-brief-writer/iteration-1
ls fixtures/mutant-brief-writer/SKILL-EVAL.md
git status --porcelain fixtures/   # SKILL-EVAL.md must NOT appear (gitignored)
```

Expected (per Phase 8): the filename-integration defect should fail Project Fit; the vague
trigger may surface only in analyst prose; the self-contradiction and dropped interview step will
likely NOT appear in SKILL-EVAL.md — that is exactly what the audit layer (step 3) exists to catch.

## Step 3 — Codex native audit, LIVE (spends API credits — human-typed only)

Optional preview first (writes prompt.txt + command-preview.sh, no Codex call):

```bash
node scripts/codex/run-native-audit.js mutant-brief-writer skill \
  --def-path fixtures/mutant-brief-writer/SKILL.md \
  --evals-path evals/fixtures/mutant-brief-writer/evals.json
```

Then the live call:

```bash
node scripts/codex/run-native-audit.js mutant-brief-writer skill \
  --def-path fixtures/mutant-brief-writer/SKILL.md \
  --evals-path evals/fixtures/mutant-brief-writer/evals.json \
  --live
```

Output: `evals/codex-runs/native-audits/skills/mutant-brief-writer/<run-id>/NATIVE-AUDIT-REPORT.md`.

**Cost note:** the per-run cost of a live native audit is currently UNKNOWN — this is the first
live run of the mode. Record the actual token/dollar cost from the Codex CLI output in the gate
results (it is an acceptance input for the plan's Phase 9 section of
`docs/evaluations/claude-code-codex-architecture-evaluation.md`).

## Step 4 — Check against ground truth (no API cost)

```bash
node scripts/run-calibration.js check
```

Defaults: eval report at `fixtures/mutant-brief-writer/SKILL-EVAL.md`, audit report = newest run
under `evals/codex-runs/native-audits/skills/mutant-brief-writer/`. Override with
`--eval-report <path>` / `--audit-report <path>` if needed.

Writes `evals/fixtures/CALIBRATION-REPORT.md` (gitignored) with the per-defect caught/missed
table. Exit 0 = 4/4, exit 1 = anything missed.

## Step 5 — Pass condition and recording

The gate PASSES when **4/4 defect classes are caught across the native + audit layers**, each
with an evidence quote:

| Defect | Expected catcher |
|--------|------------------|
| Broken output-filename integration | native metric: Project Fit (SKILL-EVAL.md) |
| Vague trigger description | native: analyst observations (SKILL-EVAL.md) |
| Internal self-contradiction | native audit: `instruction_self_consistency` |
| Silently-dropped interview step | native audit: `workflow_step_fidelity` |

The `instruction_self_consistency` and `workflow_step_fidelity` checklist items must each flag
their defect with a direct evidence quote (run-calibration.js `check` verifies this via
anchor-quote/keyword/checklist matching — see its header for the exact rules).

On PASS: record results + actual cost in the Phase 9 section of
`docs/evaluations/claude-code-codex-architecture-evaluation.md`. Phases 4+ of the plan are
unblocked.

On FAIL (any defect missed): iterate on `buildPrompt()` in `scripts/codex/run-native-audit.js`
(or the relevant native grading step), then re-run steps 3–4. Never modify the fixture. Each
iteration costs one more live audit call — record cumulative cost.

## Cleanup between full re-runs (optional)

```bash
rm -rf evals/fixtures/mutant-brief-writer/iteration-* fixtures/mutant-brief-writer/SKILL-EVAL.md
```

Keep `evals/fixtures/mutant-brief-writer/evals.json` unless you intend to regenerate (step 1 is
deterministic — regeneration produces identical scenarios for an unchanged fixture + generator).
